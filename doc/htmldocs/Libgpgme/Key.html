KeyEditCallback Method

A callback function which GPGME calls if a key edit operation is on-going.

Syntax

protected virtual int KeyEditCallback (IntPtr handle, KeyEditStatusCode status, string args, int fd)

Parameters

handle
A IntPtr handle that was provided by the user at start of operation.
status
A Libgpgme.KeyEditStatusCode.
args
A string containing the arguments.
fd
A int used for communication with the engine's key editing procedure.

Returns

A Integer. If the returning value is not 0, the key editing procedure will be stopped. See "Remarks" for more information.

Remarks

KeyEditCallback method must be overwritten by the inherited class.

It is the functions which GPGME calls if a key edit operation is on-going. The status code status and the argument line args are passed through by GPGME from the crypto engine. The int fd is 0 for normal status messages.

If status indicates a command rather than a status message, the response to the command should be written to fd. The handle was provided by the user at start of operation Key.StartEdit(Context, IntPtr, GpgmeData).

The function should return 0 or an error value. If the returning value is not 0, GPGME will stop the key editing procedure and an error will be thrown.

If the KeyEditCallback method of the inherited class throws an exception during the callback, it will be saved in the Key.LastCallbackException attribute.

StartEdit Method

Starts the key editing operation.

Syntax

protected int StartEdit (Context ctx, IntPtr handle, GpgmeData data)

Parameters

ctx
A Context used for the editing operation.
handle
A IntPtr value (handle) that can be used to identify callback calls from GPGME.
data
A GpgmeData object.

Returns

A int containing an error value. Is 0 if the operation completed successfully.

Remarks

A class that is inherited from Key can call this method in order to start a key editing operation. In this case, the Key.KeyEditCallback(IntPtr, KeyEditStatusCode, string, System.IO.Stream) method MUST be overwritten by the inherited class.
