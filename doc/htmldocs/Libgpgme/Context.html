Context Class

GPGME context for cryptographic operations.

public class Context : IDisposable

⊟ Remarks

All cryptographic operations in GPGME are performed within a context, which contains the internal state of the operation as well as configuration parameters. By using several contexts you can run several cryptographic operations in parallel, with different configuration.

⊟ DecryptAndVerify Method

Decrypts and verifys a ciphertext.

public CombinedResult DecryptAndVerify (GpgmeData cipher, GpgmeData plain)

⊟ Parameters

cipher
A source GpgmeData buffer with the ciphertext.
plain
A destination GpgmeData buffer for the plaintext (decrypted text).

⊟ Returns

A CombinedResult object with decryption and verification information.

⊟ Remarks

The function decrypts the ciphertext in the data object cipher and stores it into the data object plain. If cipher contains signatures, they will be verified.

After the operation completed, the EncryptionResult and the VerificationResult property inside the CombinedResult object can be used to retrieve more information about the signatures.

If the CombinedResult.DecryptionResult property is null, cipher did not contain any data to decrypt. However, it might still be signed. The information about detected signatures is available through the CombinedResult.VerificationResult property.

⊟ Encrypt Method

Encrypts a plaintext.

public EncryptionResult Encrypt (Key[] recipients, EncryptFlags flags, GpgmeData plain, GpgmeData cipher)

⊟ Parameters

recipients
An array of Libgpgme.Key objects that includes all recipients that shall be able to decrypt the ciphertext.
flags
Flags that have an effect on the encryption process.
plain
A source GpgmeData buffer with the plaintext.
cipher
A destination GpgmeData buffer for the ciphertext (encrypted text).

⊟ Returns

An Libgpgme.EncryptionResult object.

⊟ Remarks

The function encrypts the plaintext in the data object plain for the recipients recipients and stores the ciphertext in the data object cipher. The type of the ciphertext created is determined by the ASCII armor (Context.Armor) and text mode attributes (Context.TextMode) set in the currently used context.

If recipients is null, symmetric rather than public key encryption is performed. Symmetrically encrypted cipher text can be deciphered with Context.Decrypt(GpgmeData, GpgmeData). Note that in this case the crypto backend needs to retrieve a passphrase from the user. Symmetric encryption is currently only supported for the OpenPGP crypto backend.

⊟ SetPassphraseFunction Method

Sets a Libgpgme.PassphraseDelegate (callback function) that will be called if a crypto operation needs a passphrase from the user.

public void SetPassphraseFunction (PassphraseDelegate func)

⊟ Parameters

func
A Libgpgme.PassphraseDelegate (callback function) that will be called if a crypto operation needs a passphrase from the user.

⊟ Remarks

The function sets the function func that is used when a passphrase needs to be provided by the user. The function func needs to implemented by the user. By default, no passphrase callback function is set.

Not all crypto engines require this callback to retrieve the passphrase. It is better if the engine retrieves the passphrase from a trusted agent (a daemon process), rather than having each user to implement their own passphrase query. Some engines do not even support an external passphrase callback at all, in this case a NotSupportedException is thrown.

The user can disable the use of a passphrase callback function by calling Context.ClearPassphraseFunction() method.

⊟ Sign Method

Signs a plaintext.

public SignatureResult Sign (GpgmeData plain, GpgmeData sig, SignatureMode mode)

⊟ Parameters

plain
A source GpgmeData buffer with the plaintext.
sig
A destination GpgmeData buffer for the signature.
mode
A Libgpgme.SignatureMode that specifies the desired signature mode.

⊟ Returns

A SignatureResult object.

⊟ Remarks

The function creates a signature for the text in the data object plain and returns it in the data object sig. The type of the signature created is determined by the ASCII armor (Context.Armor) and text mode (Context.TextMode) attributes set for the context and the requested signature mode mode.

The resulting signature in the data buffer sig contains signatures for the signers listed in the current context. See Context.Signers for more information.

After the operation completed successfully, the result can be checked through the returned SignatureResult object.

⊟ Value

A Libgpgme.Context.ContextSigners object.

A signature can contain signatures by one or more signer keys. The set of keys used to create a signatures is contained in a context, and is applied to all signing operations in this context (until the set is changed).

With the Context.Signers property you can list (enumerate), add and remove (clear) all signers from the current context.

⊟ TextMode Property

Specifies if canonical text mode should be used.

public bool TextMode { set; get; }

⊟ Value

If true Canonical text mode enabled, otherwise it is disabled.

⊟ Remarks

The value of this property specifies if canonical text mode should be used. By default, text mode is not used.

Text mode is for example used for the RFC2015 signatures; note that the updated RFC 3156 mandates that the mail user agent does some preparations so that text mode is not needed anymore.

Note:

This option is only relevant to the OpenPGP crypto engine, and ignored by all other engines.

⊟ Verify Method

Verifies a signature.

public VerificationResult Verify (GpgmeData signature, GpgmeData signedtext, GpgmeData plain)

⊟ Parameters

signature
A source GpgmeData buffer with a detached or normal signature.
signedtext
A source GpgmeData buffer with the plaintext.
plain
A destination GpgmeData buffer that contains the plaintext if signature was a normal signature.

⊟ Returns

A Libgpgme.VerificationResult object.

⊟ Remarks

The function verifies that the signature in the data object signature is a valid signature. If signature is a detached signature, then the signed text should be provided in signedtext and plain should be null. Otherwise, if signature is a normal (or cleartext) signature, signedtext should be null and plain should be a writable data object that will contain the plaintext after successful verification.

The results of the individual signature verifications can be checked with the returned Libgpgme.VerificationResult object.
