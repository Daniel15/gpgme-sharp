KeyEditCallback Method

A callback function which GPGME calls if a key edit operation is on-going.

Syntax

protected override int KeyEditCallback (IntPtr handle, KeyEditStatusCode status, string args, int fd)

Parameters

handle
A IntPtr handle that was provided by the user at start of operation.
status
A Libgpgme.KeyEditStatusCode.
args
A string containing the arguments.
fd
A int used for communication with the engine's key editing procedure.

Returns

A Integer. If the returning value is not 0, the key editing procedure will be stopped. See "Remarks" for more information.

Remarks

KeyEditCallback method must be overwritten by the inherited class.

It is the functions which GPGME calls if a key edit operation is on-going. The status code status and the argument line args are passed through by GPGME from the crypto engine. The int fd is 0 for normal status messages.

If status indicates a command rather than a status message, the response to the command should be written to fd. The handle was provided by the user at start of operation Key.StartEdit(Context, IntPtr, GpgmeData).

The function should return 0 or an error value. If the returning value is not 0, GPGME will stop the key editing procedure and an error will be thrown.

If the KeyEditCallback method of the inherited class throws an exception during the callback, it will be saved in the Key.LastCallbackException attribute.

RevokeSignature Method

Creates a (key-) signature revocation certificate and adds it to the list of (key-) signatures.

Syntax

public void RevokeSignature (Context ctx, PgpRevokeSignatureOptions options)

Parameters

ctx
A Context that is used to perform the operation.
options
A PgpRevokeSignatureOptions object that contains necessary information for the (key-) signature deletion operation.

Remarks

The supplied context ctx is locked during the editing operation. Make sure that the private keys are available for all (key-) signatures that shall be revoked.
Note:

You need to specify the number(s) of the requested (key-) signature(s) in options. If you enumerate the list of (key-) signatures (UserId.Signatures property) to count and identify the correct numbers - make sure that you DON'T count "revocation" signatures/certificates. You can check this with the KeySignature.Revoked property for each (key-) signature in the list.

Example:
Example
# gpg --edit-key 321FEDCBA
Command > check
uid  Mallory <mallory@home.internal>
rev!        ABCDEF123 2009-08-01  Alice <alice@home.internal>
rev!        ABCDEF123 2009-08-02  Alice <alice@home.internal>
sig!3        321FEDCBA 2009-08-01  [self-signature]
sig!        ABCDEF123 2009-08-01  Alice <alice@home.internal>
There are four entries in this example list of (key-) signatures. The first two are "revocation signatures". That means the self-signature starts with number "1" and Alice's (key-)signature follows with number "2". (In this example, Alice created two revocation signatures/certificates. She created the first one on August 1st and the second one on August 2nd.)
